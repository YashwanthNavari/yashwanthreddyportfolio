Predictive Maintenance System

An end-to-end machine learning pipeline designed to predict equipment failures before they occur. By analyzing IoT sensor data in real-time using AWS infrastructure, this system reduced downtime by 14% in simulated environments.

System Architecture

Figure 1.1: Serverless Event-Driven Architecture on AWS

Ingestion Layer

IoT sensors transmit vibration and temperature data via MQTT to AWS IoT Core, handling over 5,000 messages per second.

Processing Layer

Lambda functions trigger on new data, normalizing inputs and running inference against the pre-trained Scikit-Learn model stored in S3.

Storage & Action

Predictions are stored in DynamoDB (Hot) and S3 (Cold). Alerts for >90% failure probability are pushed via SNS to the maintenance team.

Challenges & Solutions

warning

Handling Imbalanced Data

Failures are rare events (0.1% of data), causing the model to bias heavily towards "Normal" states.

Solution: SMOTE & Weighted Loss

Implemented Synthetic Minority Over-sampling Technique (SMOTE) to balance the training set and applied class weights during training.

speed

Lambda Cold Starts

Loading the 500MB ML model on every function invocation caused 5s+ latency for real-time inference.

Solution: Provisioned Concurrency

Used Provisioned Concurrency for Lambda and optimized model format to ONNX to reduce loading time to <200ms.
